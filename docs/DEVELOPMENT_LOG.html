术语速查（初学者）

Tavily 是什么？

Tavily 是给 AI Agent 用的联网搜索 API（不是给人类打开的搜索网站）。程序把问题发给 Tavily，返回适合喂给大模型的网页摘要。在 lanmo 里已由 app/retrieval/web_search.py 实现调用（TavilyWebSearcher）：配置 .env 的 TAVILY_API_KEY 后，当检索评估为 no_evidence 且路由 needs_web 时自动联网兜底；未配置 Key 时自动降级为空结果，不影响主流程。

常见 Agent Web 搜索工具

工具类型特点常见场景
Tavily商业 API为 LLM/RAG 优化LangChain/LangGraph 教程
SerperGoogle 结果 API接近 Google 结果需要谷歌式结果
Bing Web SearchAzure企业合规企业部署
Google Custom Search谷歌官方可限定站点垂直搜索
Exa语义搜索 API偏神经检索研究型 Agent
Brave Search API独立索引隐私向替代传统 API
DuckDuckGo社区封装免费但不稳定原型 Demo

lanmo 示例选 Tavily；以后可换 provider + app/retrieval/web_search.py 适配器（Phase 2）。

Pydantic 是什么？

Pydantic 是 Python 的数据校验与模型库（本项目用 pydantic-settings 读 .env）。定义字段与类型后，自动校验、默认值、IDE 提示；非法数据会报错。

文件用途
app/core/config.pyAppSettings：端口、日志
app/core/runtime_config.pyRuntimeConfig：Agent 运行参数
app/skills/skill_models.pySkillContext：Skill 加载结果
将来 app/schemas/ChatRequest、AgentState

步骤 20 · 路由级联：告别关键词写死 已实现

动机：纯关键词路由「换个说法就漏判」。业内成熟做法（2026 生产共识）是路由级联： 便宜的层先判、置信度不够再升级到贵的层（规则 → 语义路由 → LLM 结构化输出），把 LLM 调用压到少量模糊流量上。 详细设计见 架构 sec-07 7.2。

文件作用
app/graph/route_cascade.py级联编排：规则先判 → 置信度 < 阈值升级 LLM → 失败回退规则
app/graph/route_settings_reader.py读 app.yaml route 段：mode / 阈值 / 关键词覆盖表（不再只能改代码）
app/graph/llm_route_classifier.pyL3 层：调 Provider 按 JSON Schema 输出 RouteDecision，异常返回 None
app/graph/llm_route_prompt.py / llm_route_parser.py分类提示词（低温度、只输出 JSON）；解析容错（剥代码块、截取 JSON、字段兜底）
app/graph/route_classifier_factory.py按 mode 决定是否构建分类器；Provider 缺配置自动退化纯规则
tests/test_route_cascade.py12 个用例：配置读取、JSON 解析容错、级联三模式行为

学习要点：

  • 级联的核心是「置信度门控」：规则命中（0.6）≥ 阈值（0.6）走快路径；未命中（0.5）才问 LLM
  • LLM 路由必须永不阻塞主流程：调用失败、JSON 解析失败一律回退规则结果，reason 标注回退原因便于排查
  • L2 语义路由（示例句 embedding + 余弦相似度）需真实语义向量，待替换 HashingEmbedder 后接入
