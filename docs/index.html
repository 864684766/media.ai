lanmo 项目入门

FastAPI + Poetry  ·  更新日期：2026-07-13

用浏览器阅读文档：docs/index.html（本页）、docs/ARCHITECTURE.html、docs/VIDEO_PIPELINE.html、docs/DEVELOPMENT_LOG.html

环境要求

  • Python >= 3.10
  • Poetry >= 2.0

真实音视频生产（Stub → mp4 + TTS）

默认 config/app.yaml 已启用 local_ffmpeg + edge_tts；产出真实 mp4 与可听对白需满足下列前置（不含图生视频，clip 为 ffmpeg 色块占位 mp4）。若需 Stub 演示，将 active.clip/compose 改回 stub、tts_provider: stub。

依赖用途检查
ffmpegclip.mp4 渲染、timeline 拼接、音画混轨、字幕烧录ffmpeg -version 或设置 FFMPEG_PATH（见 .env.example）
edge-tts在线 TTS 对白（video.audio.tts_provider=edge_tts）poetry install 已含；需可访问外网
PostgreSQL分镜 / 音频资产 / compose JobDATABASE_URL + alembic upgrade head

config/app.yaml 真实音视频 preset：

video:
  providers:
    active:
      clip: local_ffmpeg
      compose: local_ffmpeg
  audio:
    tts_provider: edge_tts
    compose:
      mux_audio: true
      burn_subtitles: true

E2E 验收：poetry run python scripts/run_video_demo.py --av-mode real（断言 timeline.mp4 与非 Stub TTS）。详见 VIDEO_PIPELINE §7.5。

安装

# 创建虚拟环境并安装依赖
poetry install

# 复制环境变量文件
cp .env.example .env

双服务本地联调（media.ai + 巧克力）

Compose 仅起共享数据面；应用在宿主机各跑各的进程。不使用 Traefik / Celery / Redis。权威拓扑见 ARCHITECTURE 13.2。

写库边界：仅巧克力通过 /feed/v1/jobs 写入；media.ai 知识库 API 只读。验收：巧克力上传 project_id=demo → media-web 同 project 对话带 citation。

运行

# 推荐：启动开发服务器（带热重载）
poetry run python -m app.run

# 或使用 uvicorn 直接启动
poetry run uvicorn app.application:app --reload --host 0.0.0.0 --port 8000

# 或使用命令行脚本
poetry run lanmo

启动后访问：

  • API 文档：http://localhost:8000/docs
  • ReDoc：http://localhost:8000/redoc

入口文件说明

文件职责
app/application.py定义 FastAPI 应用对象 app，供 uvicorn 加载
app/run.py启动 uvicorn 服务器，日常开发从这里运行

PyCharm 运行配置

  1. 解释器：Settings → Project → Python Interpreter → 选择 .venv\Scripts\python.exe
  2. 运行配置：Script path 选 app/run.py；Working directory 为项目根（含 pyproject.toml）
  3. 修改代码后先 Stop 再 Run；接口未更新时 Ctrl+F5 刷新 /docs

前端 media-web

独立仓库目录 media-web/（Vite + React + Tailwind）。UI 交互一律使用 shadcn/ui 封装（src/components/ui/），禁止 window.prompt/alert/confirm 与业务层裸 button/input/select。规范见 .cursor/rules/frontend-shadcn-ui.mdc。

cd media-web
pnpm install
pnpm dev    # 默认 http://localhost:5173
pnpm run build

前端 chocolate-web

独立仓库 chocolate-web/，对接巧克力 /feed/v1。详见该目录 README.md。

cd chocolate-web
pnpm install
pnpm dev    # 默认 http://localhost:5174

POST 快速体验

# 首轮对话（服务端自动生成 conversation_id，在 message_start 事件返回）
curl -N -X POST http://localhost:8000/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "帮我写一段玄幻小说开头", "skill_id": "novel-writing"}'

体验 RAG 检索（citation 事件）

# 1. 先灌入示例小说数据（PG 必需；配置 NEO4J_URI 时同步写图谱）
poetry run python scripts/seed_novel.py --write

# 2. 问一个「设定类」问题，路由命中 needs_project 触发检索
curl -N -X POST http://localhost:8000/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "查一下主角的师父是谁", "project_id": "demo"}'
# SSE 输出中会先收到 citation 事件（chunk_id / source / 摘录），再收到正文 delta
